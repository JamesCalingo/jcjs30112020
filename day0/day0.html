Hi!

Welcome to me doing the JavaScript 30!

For those who don't know what the JavaScript 30 is, it's a 30 day coding challenge designed to teach and improve your vanilla JavaScript skills. No frameworks (React), no libraries (JQuery) - just plain ol' JavaScript.

For the people out there who don't know who I am, my name is James Calingo and I’m a developer striving to be the very best. Right now, I am looking for work, but I feel that my skills and knowledge are probably around 70-80% of where they should be due to reasons. That’s why I’m doing this challenge.

The first thing I had to do was to set up this whole page. The second thing was to try and figure out how to CSS this thing. Like I said earlier, the point of the JavaScript 30 is to learn pure JavaScript - no libraries, no frameworks, nothing. In the spirit of that, I wanted to try and create this site without the use of any Bootstrap stuff (I've been using it almost constantly since I learned it).

Designing your own CSS ain't so easy. I had to use a Scrimba course to try and create a nice blog-like atmosphere without Boostrap. I also had other things I considered for this page, but in the end, I think I'll stick with this simple CSS for now. I may change it later though.

One other thing about this whole project:
I know that this is meant to be a JavaScript project, but I've been trying to learn TypeScript as well. I'm sure most developers reading this know what that is, but for those who don't know, TypeScript is basically a stricter version of JavaScript (and that is at a VERY basic level). It's super useful to catch errors in your code BEFORE trying to run it, but due to its restrictions compared to the free-flowing JavaScript, it's tricky to incorporate it.

One of my ideas was to incorporate more TypeScript into this by making everything .ts files instead of .js files, but I decided against that.
(Slight spoilers: one of the big reasons why will be apparent tomorrow)
