DAY 10: Shift Click

View the finished product here

This one's pretty simple: click on a checkbox, hold down the shift key while checking a second checkbox (preferrably not one adjacent to the first checkbox), and watch as all the checkboxes in between get checked.

Surprisingly, the hardest part of this project was an ES6 issue: I tried to have the main function to handle the checkboxes as an arrow function, but it didn't work. Only when I went back to a regular ol' function declaration did it work. I'm not sure why that is, but again, I suspect it may be tied to the event listener.

A few other notes:

  • The app starts getting a bit funky if you play with it a bit. It's mainly due to the event being tied to the shift key behing held down, so I recommend only holding it when you're about to check a box.
  • When the page loads, the first checkbox to determine what's between the two checkboxes to mark off is the bottom of the list. In case that explanation didn't make sense (which it probably didn't), think of it this way: if you shift click the first checkbox when loading the page, it will check all of the boxes.
  • I'm not sure if other browsers do it, but when using Chrome, holding down the shift key actually shows you the last checkbox you clicked.
  • I tried figuring out a way to uncheck multiple boxes at once, but it's surprisingly tricky...
  • The magic, unfortunately, does not work on mobile (that's something I've said quite a bit during this project).
  • Apparently I REALLY need to look into what JS/TS think is available to type:Element, 'cause when using @ts-check, it's claiming that the "checked" property doesn't exist on it...
