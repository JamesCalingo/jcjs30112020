I did, however, include the arrays I'm using where the app WOULD be, though. I felt it would be neat if you guys could see what I'm using to learn this stuff. Seeing as this is pure console stuff, I wasn't sure whether or not to even make an app page, but I decided to since I wanted to show the arrays, and doing it here would probably make this page a bit ugly to look at.

As I was unsure of how I was going to do this, I actually included a script tag for the JavaScript file where I did everything on THIS page. If you go to the Console under developer tools, you'll see everything that I did. Of course, the source JS code is also visible if you check under Sources. If you do look at that, you'll notice that I console.logged empty strings in a bunch of places - this is just to separate the different tasks I did visually in the console to make it easier to go through.

This was a good refresher on Array methods, though. They're something I'm aware of but not entirely familiar with, and I feel that they're going to be important for interviews (since a lot of interviews involve arrays - granted, those are mostly for loops, but still...).

There is one thing with this, however: I'm not sure if this is just Chrome, but I did console.table for anything returning an array. However, if you load the page THEN go into dev tools, you won't see the table unless you refresh the page with the console already open. I do recommend looking at the table though - it's much prettier than just the regular console.log, even if it does require more scrolling.
