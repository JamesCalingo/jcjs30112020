These console tricks are sure to blow your mind!

So...you're probably wondering why today's "app" looks just like one of my blog posts.

That's because, like days 4 and 7, there's no app today: it's just pure JS stuff. Unlike those days, however, we're not doing anything with array methods; instead, we're going to discuss a bunch of random things you can do with the console!

For you non-JS savvy folk, the console allows a JavaScript developer to see important things: uncaught errors, variables and their types, and so much more. This is commonly done by using console.log().

However, there are a lot of things you can do with the console besides just passing in variable names and primitives (numbers, strings, and booleans).

The first one is somewhat random and obscure: if you put %s into a string you're console.logging, you can then define a value to put into the string for the %s, like this:

console.log("This is %s", "Madness!")

You're unlikely to have much need for this, however, since template literal has basically replaced concatenation as the primary method of doing stuff like this.

The next method, however, is more amusing: if you put %c at the start of whatever string you're console.logging, you can actually style it like using CSS!

console.log("%c THIS. IS. SPARTA!", "font-size: 60px")

Now, we're going to get into other console._____ methods:

  • Console.warn(), which will print a warning such as "This is a bad idea..."
  • Console.error(), which will print an error like "You can't do that genius!" You've probably seen a ton of error messages, but may not have been aware that you could actually make your own.
  • Console.info(), which will print an informative statement like "Info statements have a blue background."
  • Console.assert(), which will run a test on whatever you pass in and then produce a statement if it's false, such as 2 + 2 === 4, "Nope. Gotta love JavaScript!"
    or
    "2" + 2 === 4, "Nope. Gotta love JavaScript!"
    This one's actually kinda useful to check if your HTML elements have specific classes or ids attached to them.

I should mention that when you check each of these methods in the console except for info, they will tell you who called them and where the call was made.

There are a few others as well:

  • Console.dir() allows you to view an object of properties for a DOM element
  • Console.group() lets you group multiple console.log() statements together (I could've used this a few days ago...)
  • Console.count() shows you how many times you console.log() the same thing
  • Console.time() tells you how long a certain action takes
  • Console.table() takes an array and formats it as a table (I used this back on day 4)

There is one more method, but I don't recommend using it unless you have a VERY GOOD reason: console.clear(). You can probably guess what it does and why I don't recommend it.

So that's some things you can do with the console. Whether or not any of this is actually useful depends on your needs, but I hope you learned something from this - I certainly did. If you want to know more about any of these, you can always check w3schools or Mozilla's docs to learn more.
