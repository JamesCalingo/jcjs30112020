To wrap up today, I'd like to discuss something interesting I learned in the process of setting up this project. I mentioned this in passing on day 4, said I'd explain it later, and that "later" is now:

Originally, I had days 0-9 styled as double digits (“day00”, “day01, “day02”, etc.) If you’re at all familiar with how computers (apparently used to) function, you might know why I did this, but for those who aren’t:

When I did this, I was under the understanding that computers “alphabetize” numbers based on their first digit - no matter how long the number is. Say you had a bunch of files numbered 1-19. If you sorted these files by name, you might be surprised when it shows up as

  • 1
  • 10
  • 11
  • 12
  • and so on

instead of

  • 1
  • 2
  • 3
  • 4
  • and so on

By putting a 0 in front of the digit for single digit numbers, we can essentially “trick” the computer into thinking they’re first in order, which they’re supposed to be.

However, a little poking around showed that this wasn’t the case - the computer can count properly without taking the tens digit into account. This may be due to the fact that I have the files as “dayxx” as opposed to starting with the numbers, but I’m honestly not sure. I will note, however, that my code editor does sort my files in the method I described above, which now that I'm in double digits is both a blessing and a curse once I get to the 20s.

Also, the reason I started at 0 is because that’s where a lot of programming languages start counting for those who weren’t aware (you get used to the after a while). It also made sense to me because I wanted it to represent the sort of "pre-project" phase - not doing any actual JS30 work, just getting the environment for it ready.
