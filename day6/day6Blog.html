Y'know how search engines will try and guess what you're about to look for? This is essentially that: you enter data into the search box, and then a list of cities with text that matches what you're typing in will pop up below.

For this exercise, the data comes from This listing of the top 1,000 cities by population in the United States. I'm not sure when it was collected, but there's a line in the data that implies that it's from 2013, which means it is likely out of date (though there's another interesting quirk I'll discuss later).

This one, like the one before it, was super cool. It also taught me something new: the fetch() method!

For reasons I'd rather not get into, I haven't really done much with APIs. However, when making API calls, I'm most familiar with using AJAX (That's Asynchronous JavaScript and XML) by doing $.AJAX. As this is meant to be jQuery free, I had no idea how we would get the data going in, but learning this method of fetching data (heh) is interesting, and may have strong implications for future projects.

A few other notes:

- I ended up having an embarassingly dumb error happen where text was being taken out of the HTML. I thought I had done everything correctly (no errors were popping up), was absolutely baffled by it, and then realized that it was because I forgot a quotation mark somewhere that the error checkers were missing due to it being in a template literal. There still exists, however, a quirk where how you capitalize the text you type in will show in the results (for example, looking for something typed in lIkE tHiS will result in the results coming back capitalized like that uness it's the first letter of a word)

- You'll notice that the link to this article from the app is above the list as opposed to at the bottom. That's because if you search for something then clear the search box, the entire list pops up, and scrolling throuh 1,000 results to get back to this page isn't fun.

- This is probably one of the few instances where I just copied the example CSS. I thought it was too cool to ignore and since it wasn't featured in the video, I figured I'd just do it this way for today.

- There's a function that adds commas to the population numbers where they normally get added (i.e. for every three digits starting from the right). I considered adding it to the ranks, but given that only one of them would use it, figured it wasn't necessary.

- I mentioned earlier that there's an interesting quirk with the data, and it's this: New Jersey (where I live as of writing this) has this thing where each municipality in the state has a specific "type" (City, Township, etc.). This list, when considering "cities" in NJ, only lists the municipalities that are classified as "cities" in NJ's official records. I realized this when I found out that Edison, NJ (the fifth largest municipality in NJ and a place I used to go to a lot because I have friends who live there) was not on this list.

- Where I currently live is not on this list, but where I used to live is! I'm not saying which city it is, but I will say that it's in Massachusetts (and it's not Boston - though I did live there for the first three years of my life).

- Finally, if you peek at the link for the data, you'll see the name "Miserlou". I was tempted to make a surf rock reference somewhere in the app, but I couldn't think of a good one (though I just made one here...).
